# AI协作输入输出模板

## 一、文档目的

本文件用于约束AI处理本项目时的输入和输出格式，减少“没有边界的自由发挥”。

## 二、通用输入字段

无论哪类任务，输入至少应包含：

- 页面名称
- 页面模板类型
- 业务目标
- 关键区块
- 核心操作
- 角色/权限
- 状态集合
- 是否已有Figma参考
- 必须遵循的规范章节

## 三、任务模板一：先出页面方案

### 输入

- 页面名称
- 模板类型
- 页面目标
- 用户角色
- 业务主线位置
- 关键区块

### 输出

固定输出：

1. 页面定位
2. 页面骨架
3. 关键区块说明
4. 需要补充的状态与权限
5. 引用规范章节

禁止直接输出代码。

## 四、任务模板二：先出结构原型

### 输入

- 页面名称
- 模板类型
- 页面区块
- 关键字段
- 关键操作

### 输出

固定输出：

1. 页面结构树
2. 区块层级关系
3. 关键操作落点
4. 响应式变化点
5. 引用规范章节

## 五、任务模板三：直接出前端骨架

### 输入

- 页面名称
- 页面模板类型
- 对应Figma节点或无Figma说明
- 必须遵循的规范章节
- 是否要求响应式
- 技术栈

### 输出

固定输出：

1. 页面组件拆分
2. React结构
3. 关键样式策略
4. 状态与权限补齐点
5. 验收清单引用

禁止：

- 擅自引入第二套视觉体系
- 擅自改大圆角和控件高度
- 省略异常态与权限态

## 六、任务模板四：补齐异常态和权限态

### 输入

- 页面名称
- 已有页面结构
- 角色集合
- 缺失状态集合

### 输出

固定输出：

1. 缺失状态列表
2. 每种状态的呈现方式
3. 每种权限的显隐/禁用方案
4. 需要补到页面哪个区块

## 七、引用规范的方式

AI任务中应显式写出：

```text
必须遵循：
- 01-系统级约束总则
- 03-页面模板与骨架规范
- 04-组件与交互规则
- 05-状态权限与异常规则
- 08-页面实例映射中的对应页面
```

## 八、默认输出纪律

如果信息不全，AI应：

1. 先按页面模板输出结构
2. 显式标注默认推理规则
3. 不擅自发明视觉风格
