# 强盛进销存UI规范库

## 一、文档定位

这套规范库不是通用设计理论手册，而是当前项目可执行的页面与组件约束库。

它同时服务3类对象：

- 实现者：快速找到唯一规范来源，不再凭感觉搭页面
- AI：读取稳定输入边界，减少风格漂移和结构跑偏
- 项目文档：把页面骨架、控件规则、状态规则从业务文档里剥离出来，形成独立约束层

## 三、文件职责

| 文件 | 负责内容 |
| --- | --- |
| `00-规范使用说明` | 说明这套规范怎么用、优先级怎么判定 |
| `01-系统级约束总则` | 固定后台总体风格、框架尺寸、禁区 |
| `02-Token与代码基线` | 固定颜色、字号、间距、圆角、阴影、断点和代码实现基线 |
| `03-页面模板与骨架规范` | 固定工作台、列表、详情、新增编辑、弹层表单模板 |
| `04-组件与交互规则` | 固定控件、表格、Tabs、下拉、个人信息面板等规则 |
| `05-状态权限与异常规则` | 固定成功、失败、加载、空态、无权限、危险操作等规则 |
| `06-AI协作输入输出模板` | 固定AI任务输入与输出格式 |
| `07-AI验收清单` | 固定AI输出后的核对方式 |
| `08-页面实例映射` | 把当前页面映射到具体规范条目 |
| `spec-manifest.json` | 给AI和后续脚本读取的机器索引 |

## 四、使用原则

### 4.1 业务文档不再承载视觉细则

`00-04`主文档负责业务背景、范围、页面清单和演示主线。  
视觉、组件、状态、权限约束统一以下沉到本目录为准。

### 4.2 有冲突时的优先级

优先级从高到低：

1. 当前项目已确认的Figma节点
2. 本规范库明确写死的项目规则
3. Arco式后台默认规则
4. 实现者或AI的主观发挥

### 4.3 默认目标

默认目标不是做一个“泛化后台”，而是做出一套高信息密度、约束明确、可持续扩展的业务后台。
