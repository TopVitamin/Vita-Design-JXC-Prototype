# 组件与交互规则

## 一、文档目的

本文件统一控件与高频交互组件的结构、尺寸和使用边界。

## 二、通用控件

### 2.1 Button

- 主按钮默认高度：`32px`
- 次按钮默认高度：`32px`
- 小按钮默认高度：`28px`
- 默认小圆角
- 同一操作区中只允许一个主按钮

### 2.2 Input / Select / DatePicker

- 主输入控件默认高度：`32px`
- 文字字号优先`13/14px`
- 必填标识由label承担，不在placeholder里补“必填”
- placeholder只写输入预期，不写整句说明

### 2.3 TextArea

- 高度按内容区块决定
- 不允许默认做得过高
- 需保留字符数提示时，应放在右下或区块底部

### 2.4 Switch

- 只用于即时切换状态
- 不用于需要二次确认的危险操作

### 2.5 Checkbox
- 必须剥离原生浏览器复选框表象，尺寸严格锁定 `16x16px`，搭配微圆角 `2px`。
- 包含明显的响应反馈逻辑：悬浮时带有 `brand` 边框交互，且选中与非选中的过渡不能含混生硬。

## 三、表格类组件

### 3.1 表格列类型

当前项目允许的常见列类型：

- 复选列
- 展开列
- 链接列
- 状态列
- 人员列
- 人员+操作icon列
- 时间列
- 金额列
- 长文本列
- 操作列

### 3.2 表格规范

- 常规表头高度：`38-46px`
- 常规数据行高度：`38-46px`
- 不同明细层级允许局部使用更紧密表格
- 复杂列表必须支持横向滚动

### 3.3 分页交互
- 分页区与表格区分层，不混到表格内部
- 对于 ERP 等 B 端工具，**分页器必须高保真呈现全套交互零件**：
  - 两侧携带具备 Disabled 反馈的前后翻页箭头
  - 主分页数组容器（并伴随页码高亮状态响应）
  - 右侧跟随真实的“容量切换器（Dropdown/Select）”与真实的跳转 Input 直达枢纽

### 3.4 可交互表头排序 (Sortable Headers)
- 表头中凡具备正逆序特征的列字段必定携带排序双向箭头 `ChevronsUpDown(opacity-40)`。
- 点击表头需触发明显的三态循环流转和视觉强高亮。

## 四、筛选与切换

### 4.1 筛选区

筛选区常见组成：

- 搜索框
- 状态切换Tabs
- 快捷筛选项
- 高级筛选入口

### 4.2 Tabs / 切换器

- 用于切换视图或状态集合
- 不与普通按钮做视觉混淆
- 当前项目默认采用后台风格轻切换，不使用品牌化强视觉胶囊

## 五、信息块组件

### 5.1 提示信息块

- 用于说明填写规则、风险提示、历史说明
- 背景浅蓝、边框轻、圆角小
- 不用作成功反馈

### 5.2 详情信息块

- 由标题、标签、信息对、长文本组成
- 允许图文混合
- 优先信息对齐，不强调卡片装饰

### 5.3 步骤条

- 详情流程页可用
- 当前步骤、已完成、未完成要显式区分
- 步骤条应服务流程理解，不做装饰

## 六、个人信息下拉

### 6.1 结构

必须包含：

- 头像信息区
- 分组菜单
- 至少一个状态块、进度块或开关项
- 退出或系统级操作

### 6.2 菜单项规则

- 常规菜单项高度：`40px`
- 分组之间用分隔线
- 支持箭头进入二级能力
- 支持内嵌switch

## 七、可编辑子表格

### 7.1 适用场景

- 融资信息
- 明细录入
- 子项维护

### 7.2 规则

- 表头与静态表格一致
- 编辑行和只读行可混排
- 新增入口放表格底部
- 小屏优先横向滚动，不拆成多张独立卡片

## 八、默认推理规则

当前未单独确认但统一默认：

- 下拉菜单项高度：`32-40px`
- 个人信息面板宽度：`220-260px`
- 顶栏个人信息面板采用白底浅边框轻阴影
