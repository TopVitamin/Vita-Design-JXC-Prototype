# Token与代码基线

## 一、文档目的

本文件用于固定当前项目的视觉token与代码实现基线，避免：

- 同一项目出现多套尺寸
- AI重复发明颜色和圆角
- 代码实现和Figma口径脱节

## 二、颜色基线

### 2.1 已确认颜色

亮色模式主色与基础色以当前Figma节点为准：

- `Brand/6 = #165DFF`
- `Brand/7 = #0E42D2`
- `Text/5 = #1D2129`
- `Text/4 = #4E5969`
- `Text/3 = #86909C`
- `Fill/1 = #FFFFFF`
- `Fill/2 = #F7F8FA`
- `Fill/3 = #F2F3F5`
- `Fill/4 = #E5E6EB`
- `Line/2 = #E5E6EB`
- `Line/3 = #C9CDD4`
- `Danger/6 = #F53F3F`
- `Warning/6 = #FF7D00`
- `Success/6 = #00B42A`

### 2.2 代码token命名建议

当前原型建议保持这组语义token：

- `brand-*`
- `text-*`
- `fill-*`
- `line-*`
- `success`
- `warning`
- `danger`

禁止直接在页面里写随机HEX值，除非是插图或临时占位。

### 2.3 颜色占位符避坑 (Opacity Caveat)
由于 Tailwind CSS 的 `/{opacity}` 修饰符在遇到未规范解析的 CSS Variables 色值时会导致静默渲染失效，如果需要实现带底色透明度的微章标签（如 `StatusPill`），**绝不允许**使用如 `bg-brand-6/10` 这样的代码！必须显式声明语义对应阶梯的基础色：`bg-brand-1 text-brand-6`。

## 三、尺寸基线

### 3.1 字号

默认字号阶梯：

- 页面标题：`22-24px`
- 区块标题：`18-20px`
- 卡片/表格分组标题：`16px`
- 正文：`14px`
- 次要正文：`13px`
- 辅助说明：`12px`

### 3.2 间距

默认间距节奏：

- 微间距：`4px`
- 小间距：`8px`
- 控件内边距常用：`12px`
- 常规区块间距：`16px`
- 页面内大区块间距：`20px/24px`

### 3.3 圆角

默认圆角阶梯：

- `2px`：提示条、小色块
- `4px`：输入框、按钮、小tag
- `6px`：普通面板、小卡片
- `8px`：主容器、分组容器

禁止使用`16px+`的大圆角做后台主界面容器。

### 3.4 阴影

默认只允许两类：

- 轻阴影：用于下拉面板、浮层
- 极浅阴影：用于主容器和轻卡片

禁止用厚重投影制造层次。

## 四、响应式断点基线

建议代码层统一使用：

- `sm >= 640`
- `md >= 768`
- `lg >= 1024`
- `xl >= 1280`

规则如下：

- `lg`以上视为完整后台
- `lg`以下切抽屉侧栏
- 表格和宽子表格在`lg`以下允许横向滚动

## 五、React+Tailwind+TypeScript代码基线

### 5.1 技术基线

- React
- Tailwind CSS
- TypeScript
- 以语义组件封装基础控件

### 5.2 代码组织建议

至少拆成：

- `AppShell`
- 页面级组件
- 通用`Ui`组件
- `mock/data`
- 工具函数

### 5.3 实现原则

- 颜色和尺寸通过token表达，不在页面散写
- 布局优先由模板组件决定，不在页面里随意自由布局
- 表格、表单、浮层等高频能力优先沉淀成通用结构

## 六、默认推理规则

当前未单独确认但统一默认的值：

- 搜索框高度：`32px`
- 顶栏图标按钮尺寸：`32px`
- 主按钮默认高度：`32px`
- 小按钮默认高度：`28px`
- 列表页分页按钮高度：`32px`
